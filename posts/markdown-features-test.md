# Markdown 功能增强测试

这篇文章用来测试新增的 Markdown 插件功能。

## 📝 改进的换行处理

使用 `remark-breaks` 插件后，
普通的换行
会被正确处理，
不需要额外添加两个空格。

这样写作体验更自然，
特别适合中文内容。

## 🔧 指令语法支持

通过 `remark-directive` 插件，我们现在支持各种自定义指令语法。

## 📚 脚注功能 (GitHub Flavored Markdown)

这是一个包含脚注的段落[^1]。你可以在文中添加多个脚注[^2]。

脚注可以包含更复杂的内容[^complex]。

注意：脚注功能现在由 `remark-gfm` 提供，符合 GitHub Flavored Markdown 规范。

[^1]: 这是第一个脚注的内容。
[^2]: 这是第二个脚注，可以包含**粗体文字**和*斜体文字*。
[^complex]: 这个脚注包含复杂内容：

    -   列表项目 1
    -   列表项目 2

    还可以包含[链接](https://example.com)和`代码`。
